# let'sgetpattern - 音乐制作人日程规划助手

一个专为音乐制作人设计的日程规划和进度跟踪工具，帮助制作人高效管理音乐制作项目。

## 功能特点

### 📅 日程规划模块

- 🎵 **歌曲管理**：添加、编辑、删除歌曲，跟踪每首歌的制作进度
- 📊 **进度跟踪**：实时显示任务完成情况和整体进度
- ⏱️ **时间管理**：跟踪每首歌的预计时长和已用时长，内置计时器功能
- 📅 **时间线规划**：自动计算每天需要的工作时长，合理分配时间
- 📊 **甘特图**：可视化展示项目进度和时间安排
- 📈 **项目进度视图**：多维度查看项目整体完成情况
- ✅ **任务清单**：10个标准制作流程任务，确保不遗漏任何步骤
- 📈 **统计信息**：一目了然的完成情况统计（剩余天数、已完成、进行中、总进度）
- 💡 **编曲/混音小知识**：随机展示制作技巧和知识点
- 📅 **日期管理**：设置项目开始日期，自动计算剩余天数
- 💾 **数据备份**：支持数据导入导出，方便备份和迁移

### 🎵 音乐理论模块

- 🎼 **五度圈**：可视化五度圈，包含大调和关系小调
- 🎹 **常见和弦级数套路**：12个调性的常用和弦进行参考
- 📚 **和弦知识速查**：
  - 基础和弦类型详解
  - 各调和弦级数表
  - 和弦进行示例
- 🎶 **中古调式构成音速查**：快速查询各调式在不同根音下的构成音

## 制作流程

每首歌的制作包含以下10个任务：

1. 新曲风前期准备（至少一周）：听10张专辑，搜索曲风介绍（BPM、配器音色、代表人物、经典曲目、最新曲目）
2. 确定子曲风，找到3首参考歌，发给队长
3. 制作Demo，自己觉得OK
4. Demo发给队长，获取意见
5. 完成编曲
6. 编曲发给队长，获取意见
7. 编曲OK，开始混音母带
8. 混音母带发给队长，获取意见
9. 队长OK，发给校长
10. 校长OK，完成制作

## 时间分配

- **每首歌预计时长**：40 有效小时（可自定义，范围 20-60 小时；初期建议多花些时间）
- **每天最多工作时长**：6 有效小时
- **推荐每天工作时长**：2 小时（已养成习惯）
- **完成周期**：每天 2 小时则 20 天完成一首歌

## 使用方法

### 基本操作

1. 直接在浏览器中打开 `index.html` 文件
2. 首次使用建议先设置项目开始日期（点击"📅 设置开始日期"按钮）
3. 点击"添加新歌"按钮创建第一首歌
4. 填写歌曲信息，包括：
   - 歌曲名称
   - 子曲风（建议：90%精力用于制作你当下正喜欢的曲风）
   - 预计时长（默认40小时，范围20-60小时）
   - 是否为新曲风（新曲风需要至少一周前期准备）
   - 当前阶段（曲风研究、Demo制作、编曲、混音母带、队长审核、校长审核、已完成）
   - 已完成的任务（勾选复选框）
   - 已用时长（可使用内置计时器）
   - 备注
5. 保存后即可在列表中查看和管理

### 功能使用

- **时间线规划**：在"日程规划"标签页查看时间线视图和项目进度视图
- **甘特图**：可视化查看所有歌曲的时间安排和进度
- **计时器**：在歌曲卡片上点击计时按钮，开始跟踪实际工作时长
- **音乐理论**：切换到"音乐理论"标签页，查看五度圈、和弦进行、调式等参考资料
- **数据备份**：定期使用"导出备份"功能保存数据，需要时可"导入备份"恢复

### 自动计算

系统会自动计算：
- 每首歌的完成进度
- 剩余工作时间
- 每天建议工作时长
- 整体项目进度
- 剩余天数（基于开始日期和总天数180天）

## 数据存储

所有数据存储在浏览器的 localStorage 中，无需服务器，完全本地运行。

## 项目设置

- **总天数**：180 天（从开始日期算起）
- **目标歌曲数**：9 首
- **剩余天数**：根据设置的开始日期自动计算

## 技术栈

- **HTML5**：语义化结构
- **CSS3**：现代渐变设计，响应式布局
- **原生 JavaScript**：无依赖，纯前端实现
- **localStorage**：本地数据存储

## 浏览器支持

支持所有现代浏览器（Chrome, Firefox, Safari, Edge）
